Frequently Asked Questions

What is the difference between var, let, and const?

var

  • Function scoped (ব্লক স্কোপ না)
  • Same variable can be redeclared and updated
  • Hoisted to the top of its scope

let

  • Block scoped ({} এর ভিতরে সীমাবদ্ধ)
  • Can be updated but not redeclared in the same scope
  • Hoisted but not initialized (ReferenceError if accessed before declaration)

const

  • Block scoped
  • Cannot be updated or redeclared
  • Must be initialized at the time of declaration
  • For objects/arrays, properties can be changed, but the reference can’t
what is the difference between map(), forEach(), and filter()?

map()

map() একটি নতুন অ্যারে রিটার্ন করে, যেখানে প্রতিটি উপাদান কিছু পরিবর্তন বা প্রসেসের মাধ্যমে তৈরি হয়। 


            const numbers = [1, 2, 3];
            const doubled = numbers.map(num => num * 2);
            // doubled: [2, 4, 6]

forEach()

forEach() শুধুমাত্র প্রতিটি উপাদানের উপর কাজ করে, কিন্তু কোনো নতুন অ্যারে রিটার্ন করে না। 


            const numbers = [1, 2, 3];
            numbers.forEach(num => {
              console.log(num * 2);
            });
            // Output: 2, 4, 6 (but no return)

filter()

filter() একটি নতুন অ্যারে রিটার্ন করে, যেখানে শুধুমাত্র সেই উপাদানগুলো থাকে যেগুলো একটি শর্ত পূরণ করে। 


            const numbers = [1, 2, 3, 4];
            const even = numbers.filter(num => num % 2 === 0);
            // even: [2, 4]
Explain arrow functions and how they are different from regular functions.

Arrow Function হলো JavaScript ES6 এ আসা একটি নতুন ধরনের ফাংশন লেখার shorthand (সংক্ষিপ্ত) উপায়।

Arrow Function Syntax: 

            const add = (a, b) => {
              return a + b;
            }

Regular Function Syntax: 

            function add(a, b) {
              return a + b;
            }

মূল পার্থক্য:

  • Syntax: Arrow function লেখা সহজ এবং সংক্ষিপ্ত।
  • this কীওয়ার্ড: Arrow functions তাদের parent scope থেকে this ইনহেরিট করে, যেখানে regular functions এ this আলাদা behave করে।
  • Constructor: Arrow functions দিয়ে constructor তৈরি করা যায় না, কিন্তু regular functions দিয়ে যায়।
  • Hoisting: Regular functions hoisting support করে, কিন্তু arrow functions করে না।

উদাহরণ: 

            // Arrow Function
            const greet = (name) => console.log(`Hello, ${name}`);
            
            // Regular Function
            function greet(name) {
              console.log(`Hello, ${name}`);
            }
How JavaScript Promises work?

A Promise is a JavaScript object that represents the eventual completion (or failure) of an asynchronous operation and its resulting value.

States of a Promise:

  • Pending: Initial state, neither fulfilled nor rejected.
  • Fulfilled: The operation completed successfully.
  • Rejected: The operation failed.

Basic Example: 

            // Creating a Promise
            const myPromise = new Promise((resolve, reject) => {
              const success = true;
            
              if (success) {
                resolve("Operation successful!");
              } else {
                reject("Something went wrong.");
              }
            });
            
            // Using the Promise
            myPromise
              .then(result => {
                console.log(result); // Output: Operation successful!
              })
              .catch(error => {
                console.error(error);
              });

Chaining Promises: 

            fetch("https://api.example.com/data")
              .then(response => response.json())
              .then(data => {
                console.log(data);
              })
              .catch(error => {
                console.error("Error fetching data:", error);
              });

Promises help manage asynchronous tasks more easily and avoid deeply nested callbacks (callback hell).

How closures work in JavaScript?

A closure is created in JavaScript when a function is defined inside another function and the inner function accesses variables from the outer function’s scope. Even after the outer function has finished executing, the inner function still has access to those variables.

Example:


            // Outer function
            function outerFunction() {
              let count = 0;
            
              // Inner function (closure)
              function increment() {
                count++;
                console.log(count);
              }
            
              return increment;
            }
            
            const counter = outerFunction();
            counter(); // 1
            counter(); // 2
            counter(); // 3

Here, increment is a closure. It remembers the value of count from outerFunction, even after the outer function has returned. This is possible because closures "close over" the variables they need from their outer scope.
